# Лабораторная работа №2 (АОИС)

В этой лабораторной реализован анализ булевой функции, заданной в текстовом виде.  
Программа сама разбирает выражение, строит таблицу истинности и по ней выполняет все дальнейшие преобразования и проверки.

## Что умеет программа

1. Принимает логическую формулу в свободной записи.
2. Строит таблицу истинности.
3. Строит СДНФ и СКНФ.
4. Выводит числовые формы для СДНФ и СКНФ.
5. Выводит индексную форму функции (вектор и индекс).
6. Определяет принадлежность к классам Поста (`T0`, `T1`, `S`, `M`, `L`).
7. Строит полином Жегалкина.
8. Находит фиктивные переменные.
9. Считает булевы производные:
- частные;
- смешанные (порядки от 1 до 4).
10. Минимизирует функцию тремя способами:
- расчетный метод (со стадиями склеивания и проверкой лишних импликант);
- расчетно-табличный метод (со стадиями склеивания и таблицей простых импликант);
- табличный метод (карта Карно) с выводом карты и групп.

## Поддерживаемые операции в формуле

- отрицание: `!` (также можно `¬`);
- конъюнкция: `&` (также можно `∧`);
- дизъюнкция: `|` (также можно `∨`);
- импликация: `->` (также можно `→`);
- эквивалентность: `~` (также можно `↔`, `≡`);
- скобки: `(` `)`;
- константы: `0`, `1`.

Пример корректного ввода:

```text
!(!a→!b)∨c
```

## Краткая теория

1. **Таблица истинности**  
Для всех наборов значений переменных (`0/1`) вычисляется значение функции.  
Это базовый объект, из которого уже строятся канонические формы, индекс, производные и минимизация.

2. **СДНФ и СКНФ**  
СДНФ — дизъюнкция элементарных конъюнкций для строк, где `f = 1`.  
СКНФ — конъюнкция элементарных дизъюнкций для строк, где `f = 0`.

3. **Числовые формы**  
Для СДНФ перечисляются номера наборов, на которых функция равна `1` (минтермы).  
Для СКНФ — номера наборов, на которых функция равна `0` (макстермы).

4. **Индексная форма**  
Значения функции по строкам таблицы записываются как двоичный вектор.  
Этот же вектор можно трактовать как целое число (индекс функции).

5. **Классы Поста**  
Проверяются пять свойств:
- `T0` — сохраняет 0;
- `T1` — сохраняет 1;
- `S` — самодвойственность;
- `M` — монотонность;
- `L` — линейность (по полиному Жегалкина).

6. **Полином Жегалкина**  
Представление булевой функции в базисе XOR/AND над полем GF(2).  
Коэффициенты вычисляются по вектору функции через треугольное преобразование.

7. **Фиктивная переменная**  
Переменная считается фиктивной, если изменение только этой переменной не меняет значение функции.

8. **Булева производная**  
Производная по переменной `x`:  
`∂f/∂x = f(x=0,...) ⊕ f(x=1,...)`
Смешанные производные строятся повторно по набору переменных.

9. **Минимизация**  
- В расчетном методе выполняется поэтапное склеивание импликант и удаление лишних.  
- В расчетно-табличном — после склеивания строится таблица покрытия минтермов простыми импликантами.  
- В методе Карно используется карта Грея и группировка единиц в прямоугольники размеров \(2^k\).

## Структура проекта

- `src/expression_parser.py` — разбор логического выражения и AST.
- `src/truth_table.py` — построение таблицы истинности.
- `src/canonical_forms.py` — СДНФ/СКНФ, числовые и индексная формы.
- `src/post_classes.py` — проверка классов Поста.
- `src/zhegalkin.py` — полином Жегалкина.
- `src/derivatives.py` — фиктивные переменные и булевы производные.
- `src/minimization.py` — расчетный и расчетно-табличный методы.
- `src/karnaugh.py` — карта Карно.
- `src/analyzer.py` — общий сценарий анализа.
- `src/report.py` — форматирование отчета.
- `src/main.py` — точка входа CLI.
- `tests/` — модульные тесты.

## Как запустить программу

Из корня проекта:

```powershell
python -m src.main
```

После запуска введите формулу в консоль.

Можно передать формулу сразу аргументом:

```powershell
python -m src.main "!(!a->!b)|c"
```

## Замечания и ограничения

1. Максимум 5 переменных (`a..e`).
2. Если переменных нет (константная функция), программа корректно обрабатывает этот случай.
3. Для больших СДНФ/СКНФ (например, 5 переменных и много единиц) отчет получается объемным — это нормально.
4. Для корректного парсинга импликации используйте `->` (или символ `→`).
