# Лабораторная работа №1 (АОИС)

В этой работе реализованы операции с числами на уровне битов.  
Ограничение по условию соблюдено: данные хранятся в массивах из `0` и `1` длиной 32 бита, без использования готовых средств перевода между форматами.

## Что делает программа

1. Переводит целое число из десятичного формата в:
- прямой код;
- обратный код;
- дополнительный код.

2. Выполняет операции над целыми числами:
- сложение в дополнительном коде;
- вычитание через отрицание вычитаемого и сложение;
- умножение в прямом коде;
- деление в прямом коде как число с фиксированной двоичной точкой.

3. Выполняет операции с float32 (IEEE-754, 32 бита):
- сложение;
- вычитание;
- умножение;
- деление.

4. Складывает числа в BCD-коде `Excess-3`.

Для каждой операции выводится результат в двоичном и десятичном виде.

## Краткая теория

1. Прямой код  
Старший бит это знак, остальные биты это модуль числа.

2. Обратный код  
Для отрицательных чисел получается инверсией битов положительного представления.

3. Дополнительный код  
Получается из обратного добавлением единицы.  
Этот код удобен для сложения и вычитания, потому что операция сводится к обычному сложению битовых массивов.

4. IEEE-754 (float32)  
Формат: `1 бит знака + 8 бит порядка + 23 бита мантиссы`.  
В проекте реализованы ручные кодирование и декодирование значения в этот формат.

5. Excess-3  
Каждая десятичная цифра хранится как `(цифра + 3)` в 4 битах.  
При сложении тетрад выполняется корректировка, чтобы результат оставался валидным.

## Структура проекта

- `src/bit_core.py` — базовые битовые операции и служебные функции.
- `src/integer_codes.py` — прямой, обратный, дополнительный коды.
- `src/integer_arithmetic.py` — арифметика целых чисел.
- `src/float32_ieee.py` — операции и преобразования IEEE-754 float32.
- `src/excess3.py` — кодирование/декодирование и сложение Excess-3.
- `src/main.py` — консольный запуск программы.
- `tests/test_binary_lab.py` — юнит-тесты.

## Как запустить

```

```

## Ограничения

1. Целые числа обрабатываются в рамках 32-битного представления.
2. Для `Excess-3` в 32 битах поддерживается до 8 десятичных цифр.
3. Деление в прямом коде хранит модуль без десятичного масштабирования.
   Дробная часть строится как fixed-point представление, а `precision` задаёт максимум двоичных знаков после точки, которые удалось сохранить в 31 битах модуля.
