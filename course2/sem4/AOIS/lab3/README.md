# Лабораторная работа №3 (АОИС)

В проекте реализован синтез и минимизация цифровых схем по трём заданиям:
комбинационные схемы (ОДС-3 и преобразователь кода 2421 со смещением) и цифровой автомат (счётчик mod 8 на T-триггерах).

## Что делает программа

1. Задание 1 (ОДС-3):
- строит таблицу истинности на 8 наборов (`X1, X2, X3 -> S, C`);
- формирует СКНФ для `S` и `C`;
- минимизирует функции;
- выводит описание схем в базисе `NOT/AND/OR`.

2. Задание 2 (код 2421, смещение `n = 6`):
- использует таблицу кодирования цифр `0..9` в коде 2421;
- строит таблицу преобразования `d -> (d + 6) mod 10`;
- обрабатывает неиспользуемые входные наборы как `don't care`;
- минимизирует 4 выходные функции (`Y3..Y0`);
- выводит схему в базисе `NOT/AND/OR`.

3. Задание 3 (цифровой автомат):
- строит таблицу переходов для состояний `Q2 Q1 Q0`;
- реализует переход `(Q + 1) mod 8`;
- вычисляет функции возбуждения T-триггеров (`T0, T1, T2`);
- минимизирует их;
- выводит комбинационную часть схемы в базисе `NOT/AND/OR`.

## Краткая теория

1. Таблица истинности  
Это полный набор входных комбинаций и соответствующих им значений функции.

2. СКНФ (совершенная конъюнктивная нормальная форма)  
СКНФ строится по строкам, где функция равна `0`: для каждой такой строки формируется дизъюнкт, а затем все дизъюнкты перемножаются.

3. Минимизация логических функций  
В проекте используется минимизация вида СДНФ:
- строятся допустимые импликанты (с учётом `don't care`);
- выбирается покрытие всех единичных наборов;
- оптимизация идёт сначала по числу термов, затем по числу литералов.

4. Базис НЕ, И, ИЛИ  
Любое минимизированное выражение представляется как список логических элементов `NOT`, `AND`, `OR` и связей между ними.

5. T-триггер  
Для перехода состояния используется формула возбуждения:
`T = Q xor Q_next`.  
По таблице переходов получаются функции `T0, T1, T2`, которые затем минимизируются.

## Структура проекта

- `main.py` — точка входа, печатает общий отчёт по всем заданиям.
- `src/app.py` — сборка и форматирование итогового вывода.
- `src/common/boolean.py` — таблицы, СКНФ, минимизация, проверка выражений.
- `src/common/circuit.py` — синтез схемы в базисе `NOT/AND/OR`.
- `src/common/render.py` — вывод таблиц и секций.
- `src/part1/full_adder.py` — задание 1 (ОДС-3).
- `src/part2/bcd2421_offset.py` — задание 2 (2421 + смещение 6).
- `src/part3/counter_tff.py` — задание 3 (счётчик mod 8 на T-триггерах).
- `tests/` — юнит-тесты.

## Ограничения и примечания

1. В проекте реализованы именно варианты из задания: ОДС-3, код 2421 со смещением 6, счётчик mod 8.
2. Алгоритм минимизации рассчитан на функции малой размерности (что и требуется для лабораторной).
3. `don't care` участвуют в минимизации, но не считаются обязательными для покрытия.
