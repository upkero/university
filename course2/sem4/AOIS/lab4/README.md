# Лабораторная работа №4 (АОИС)

В этой работе реализована хеш-таблица с открытой адресацией.  
Коллизии решаются линейным поиском (linear probing).

Хеш-функция:

`h(V) = V mod H + B`

где:
- `V` — числовой ключ;
- `H` — размер таблицы;
- `B` — базовый (начальный) адрес.

## Что реализовано

1. CRUD-операции:
- `create(key, value)` — добавить новую пару ключ-значение;
- `read(key)` — получить значение по ключу;
- `update(key, value)` — обновить значение;
- `delete(key)` — удалить запись.

2. Разрешение коллизий:
- при занятости ячейки выполняется последовательный переход к следующей;
- в конце таблицы поиск продолжается с начала (циклически).

3. Обработка типовых ситуаций:
- запрет дублирования ключей;
- ошибка при переполнении таблицы;
- корректная работа после удалений (через пометки удалённых слотов).

4. Просмотр внутреннего состояния:
- метод `dump()` возвращает снимок всех слотов таблицы;
- в CLI выводится адрес, статус и значение.

5. Авторасширение:
- при переполнении таблица увеличивается в 2 раза (если включено).

## Краткая теория

1. Идея хеш-таблицы  
Ключ преобразуется в адрес ячейки через хеш-функцию.  
Это даёт быстрый доступ к данным в среднем случае.

2. Коллизии  
Коллизия возникает, когда разные ключи дают один и тот же адрес.  
В этой работе используется линейное пробирование: если место занято, берётся следующая позиция.

3. Почему нужны пометки удалённых элементов  
Если после удаления просто поставить `None`, можно «сломать» цепочку поиска.  
Поэтому удалённые элементы помечаются специальным маркером, чтобы поиск оставался корректным.

## Как использовать

Пример работы в коде:

```python
from src import HashTable

table = HashTable(size=20, base=0)

table.create(10, "десять")
table.create(30, "тридцать")  # коллизия с 10 при H=20

print(table.read(10))         # десять

table.update(10, "10")
print(table.read(10))         # 10

removed = table.delete(30)
print(removed)                # тридцать
```

Пример с авторасширением:

```python
table = HashTable(size=2, base=0, auto_resize=True)
table.create(1, "one")
table.create(2, "two")
table.create(3, "three")  # таблица расширится автоматически
```

## Как запустить CLI

Из корня проекта:

```powershell
python -m src.main
```

После запуска доступно меню:
- `1` — добавить запись;
- `2` — найти запись по ключу;
- `3` — обновить запись;
- `4` — удалить запись;
- `5` — показать текущее количество элементов;
- `6` — показать содержимое таблицы;
- `0` — завершить программу.

При запуске CLI также запрашивается режим авторасширения.

## Ограничения

1. Ключи поддерживаются только целочисленные (`int`).
2. Авторасширение опционально: можно включить при создании таблицы.
3. `read` возвращает `None`, если ключ не найден.
